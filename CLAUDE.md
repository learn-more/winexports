# WinExports

Static site listing Windows DLL exports across NT versions, deployed to GitHub Pages via Hugo.
The Hugo templates are based on the [apisets](https://apisets.info) repository.

## Running Python scripts

Always run Python scripts with `uv` — it handles inline PEP 723 dependencies automatically:

    uv run scripts/01-extract_dlls.py [--force]
    uv run scripts/02-dump_exports.py <input_directory>
    uv run scripts/03-generate_hugo_content.py

## Pipeline

**Step 1 — Extract DLLs from ISOs** (`scripts/01-extract_dlls.py`)
- Scans `ISO_DIR` for `.iso` files and extracts System32 files into `OUTPUT_DIR`
- Handles both WIM/ESD (Vista+) and I386/DL_ (XP/Server 2003) layouts
- Requires 7-Zip; configure `ISO_DIR`, `OUTPUT_DIR`, and `SEVENZIP` at the top of the script
- Uses stdlib only — `uv run` works but plain `python` is fine too

**Step 2 — Dump exports** (`scripts/02-dump_exports.py`)
- Takes a directory of `ntNN_ARCH` subdirectories, each containing DLL files
- Reads PE export tables using `pefile` and writes one JSON per DLL to `data/exports/{version}/`
- Only DLLs listed in `scripts/include_files.txt` are processed

**Step 3 — Generate Hugo content** (`scripts/03-generate_hugo_content.py`)
- Aggregates per-version DLL JSONs into `data/dlls/`
- Generates `content/dlls/*.md` (one page per DLL)
- Builds `static/data/function_names.json` (function name → URL, for client-side search)
- Builds `static/data/fn/*.json` (compact cross-reference data for multi-DLL functions)
- Generates `data/versions.json` and `content/_index.md`

Steps 1 and 2 are run locally only — their output (`data/exports/`) is committed to the repo.
Step 3 and the Hugo build run automatically on CI (`.github/workflows/deploy.yml`) on every push to master.

**Step 4 — Hugo build**
- Run `hugo` (or `hugo --minify` for production)
- CI/CD: GitHub Actions runs step 3 then builds and deploys on push to master

## Auto-generated files — do not edit manually

- `data/dlls/` — regenerated by 03-generate_hugo_content.py
- `content/dlls/` — regenerated by 03-generate_hugo_content.py
- `static/data/fn/` — regenerated by 03-generate_hugo_content.py
- `static/data/function_names.json` — regenerated by 03-generate_hugo_content.py
- `data/versions.json` — regenerated by 03-generate_hugo_content.py
- `content/_index.md` — regenerated by 03-generate_hugo_content.py

## Hand-maintained files

- `content/functions/_index.md` and `content/functions/fn.md` — preserved across regeneration
- `layouts/shortcodes/` — Hugo shortcodes for DLL tables and function search
- `layouts/functions/single.html` — client-side rendered per-function detail page
- `scripts/include_files.txt` — controls which DLLs are extracted
- `static/css/kraiklyn-override.css` — theme overrides
